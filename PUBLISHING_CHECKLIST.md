# Publishing Checklist

Use this before pushing the portfolio package to a public GitHub repository.

## Account

- Enable multi-factor authentication on GitHub.
- Use a professional profile name, photo or avatar, and short bio.
- Add a concise profile line such as: `Network automation and Python workflow tooling`.

## Repository

- Recommended name: `network-refresh-toolkit`
- Recommended visibility for first review: private
- Recommended public description:
  - `Sanitized Python tooling examples for network switch refresh workflows.`

## File Review

Confirm the repository does not contain:

- real hostnames
- real IP addresses
- private domains
- raw configs
- raw logs
- screenshots from private systems
- API keys or secrets
- generated executables
- build folders
- cache folders

## Suggested First Commit

```text
Initial sanitized network refresh toolkit portfolio
```

## Suggested README Topics

- what problem the tools solve
- why operator review matters
- what safety boundaries exist
- what example artifacts look like
- what future work remains

## After Publishing

- Open the repo in an incognito/private browser window.
- Click every example link.
- Confirm the SVG renders correctly.
- Confirm no private names or paths are visible.
- Pin the repository on the GitHub profile if it looks polished.
